## Clone and Install

```bash
git clone https://github.com/franciscomoretti/chatjs.git
cd chatjs
bun install
```

## Environment Setup

```bash
cp .env.example .env.local
```

<Tabs>
  <Tab title="Vercel (Recommended)">
    [![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/franciscomoretti/chatjs)

    | `VERCEL_URL` | System |
    | `VERCEL_PROJECT_PRODUCTION_URL` | System |

    | Variable | Description |

### Optional Variables

These work on both Vercel and self-hosted:

| Variable              | Description          |
| --------------------- | -------------------- |
| `CRON_SECRET`         | Cleanup cron job     |
| `REDIS_URL`           | Resumable streams    |
| `OPENAI_API_KEY`      | Direct OpenAI access |
| `TAVILY_API_KEY`      | Web search           |
| `EXA_API_KEY`         | Web search           |
| `FIRECRAWL_API_KEY`   | Web scraping         |
| `LANGFUSE_PUBLIC_KEY` | Observability        |
| `LANGFUSE_SECRET_KEY` | Observability        |
| `LANGFUSE_BASE_URL`   | Custom Langfuse URL  |
| `MCP_ENCRYPTION_KEY`  | MCP connectors       |

## Database Setup

```bash
bun db:migrate
```

<Accordion title="Development with Neon Branching">
  When working on schema changes, use Neon branching to avoid affecting production:

```bash
# First time: authenticate
bunx neonctl auth
bunx neonctl set-context --project-id <your-project-id>

# Quick workflow
bun db:branch:start    # Create dev-local branch
bun dev                # Uses branch DB
bun db:generate
bun db:migrate
bun db:branch:stop     # Delete branch, switch to main
```

| Command                       | Description                   |
| ----------------------------- | ----------------------------- |
| `bun db:branch:start [name]`  | Create and switch to branch   |
| `bun db:branch:stop [name]`   | Delete branch, switch to main |
| `bun db:branch:use [name]`    | Switch to branch              |
| `bun db:branch:create [name]` | Create branch only            |
| `bun db:branch:delete [name]` | Delete branch only            |
| `bun db:branch:list`          | List all branches             |

</Accordion>

## Run Development Server

```bash
bun dev
```

Open [http://localhost:3000](http://localhost:3000) to start building.
