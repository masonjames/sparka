## Clone and Install

```bash
git clone https://github.com/franciscomoretti/chatjs.git
cd chatjs
bun install
```

## Environment Setup

```bash
cp .env.example .env.local
```

<Tabs>
  <Tab title="Vercel (Recommended)">
    ### Deploy with Vercel

    Use the deploy button to set up with Vercel Postgres and Blob integrations:

    [![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/franciscomoretti/chatjs)

    After deploy, pull environment variables locally:

    ```bash
    vercel link
    vercel env pull .env.local
    ```

    ### Provided Automatically
    - `VERCEL_OIDC_TOKEN` — replaces `AI_GATEWAY_API_KEY`
    - `DATABASE_URL` — via Vercel Postgres
    - `BLOB_READ_WRITE_TOKEN` — via Vercel Blob

    ### Required Variables
    - `AUTH_SECRET` — [Generate one](https://generate-secret.vercel.app/32)
    - One auth provider pair:
      - `AUTH_GITHUB_ID` + `AUTH_GITHUB_SECRET`
      - `AUTH_GOOGLE_ID` + `AUTH_GOOGLE_SECRET`
  </Tab>
  <Tab title="Self-hosted">
    ### Required Variables

    ```bash
    DATABASE_URL=postgres://...
    BLOB_READ_WRITE_TOKEN=...
    AUTH_SECRET=...
    AI_GATEWAY_API_KEY=...  # or VERCEL_OIDC_TOKEN
    ```

    ### Auth Provider (choose one)

    ```bash
    # GitHub
    AUTH_GITHUB_ID=...
    AUTH_GITHUB_SECRET=...

    # Or Google
    AUTH_GOOGLE_ID=...
    AUTH_GOOGLE_SECRET=...
    ```

    ### Optional

    ```bash
    REDIS_URL=redis://...           # Resumable streams
    OPENAI_API_KEY=...              # Direct OpenAI access
    TAVILY_API_KEY=...              # Web search
    EXA_API_KEY=...                 # Web search
    FIRECRAWL_API_KEY=...           # Web scraping
    LANGFUSE_PUBLIC_KEY=...         # Observability
    LANGFUSE_SECRET_KEY=...
    ```
  </Tab>
</Tabs>

## Database Setup

```bash
bun db:migrate
```

<Accordion title="Development with Neon Branching">
  When working on schema changes, use Neon branching to avoid affecting production:

  ```bash
  # First time: authenticate
  bunx neonctl auth
  bunx neonctl set-context --project-id <your-project-id>

  # Quick workflow
  bun db:branch:start    # Create dev-local branch
  bun dev                # Uses branch DB
  bun db:generate
  bun db:migrate
  bun db:branch:stop     # Delete branch, switch to main
  ```

  | Command | Description |
  |---------|-------------|
  | `bun db:branch:start [name]` | Create and switch to branch |
  | `bun db:branch:stop [name]` | Delete branch, switch to main |
  | `bun db:branch:use [name]` | Switch to branch |
  | `bun db:branch:list` | List all branches |
</Accordion>

## Run Development Server

```bash
bun dev
```

Open [http://localhost:3000](http://localhost:3000) to start building.
