    ## Deploy to Vercel

    [![Deploy with Vercel](https://vercel.com/button)](https://vercel.com/new/clone?repository-url=https://github.com/franciscomoretti/chatjs)

    This creates a new repo in your GitHub account and deploys it to Vercel.

    ### Configure Environment Variables

    In the Vercel dashboard, add these variables:

    | Variable | Description |
    |----------|-------------|
    | `AUTH_SECRET` | [Generate one](https://generate-secret.vercel.app/32) |

    **Auth Provider (choose one pair):**
    | Provider | Variables |
    |----------|-----------|
    | GitHub | `AUTH_GITHUB_ID` + `AUTH_GITHUB_SECRET` |
    | Google | `AUTH_GOOGLE_ID` + `AUTH_GOOGLE_SECRET` |
    | Vercel | `VERCEL_APP_CLIENT_ID` + `VERCEL_APP_CLIENT_SECRET` |

    <Note>
      Vercel automatically provides `DATABASE_URL`, `BLOB_READ_WRITE_TOKEN`, `VERCEL_OIDC_TOKEN`, and other system variables.
    </Note>

    ### Local Development

    Clone your new repo and pull environment variables:

    ```bash
    git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git
    cd YOUR_REPO
    bun install
    vercel link
    vercel env pull .env.local
    ```

    Remove the docs folder (you won't need it):

    ```bash
    rm -rf docs
    ```

  <Tab title="Manual Setup">
    ## Create Your Project

    Click **"Use this template"** on the [ChatJS repo](https://github.com/franciscomoretti/chatjs), then clone your new repo:

    ```bash
    git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git
    cd YOUR_REPO
    bun install
    ```

    Remove the docs folder (you won't need it):

    ```bash
    rm -rf docs
    ```

    ### Configure Environment Variables

    ```bash
    cp .env.example .env.local
    ```

    | Variable | Description |
    |----------|-------------|
    | `DATABASE_URL` | PostgreSQL connection string |
    | `BLOB_READ_WRITE_TOKEN` | Vercel Blob token |
    | `AUTH_SECRET` | [Generate one](https://generate-secret.vercel.app/32) |
    | `AI_GATEWAY_API_KEY` | AI Gateway API key |

    **Auth Provider (choose one pair):**
    | Provider | Variables |
    |----------|-----------|
    | GitHub | `AUTH_GITHUB_ID` + `AUTH_GITHUB_SECRET` |
    | Google | `AUTH_GOOGLE_ID` + `AUTH_GOOGLE_SECRET` |
    | Vercel | `VERCEL_APP_CLIENT_ID` + `VERCEL_APP_CLIENT_SECRET` |

  </Tab>

### Optional Variables

| Variable              | Description          |
| --------------------- | -------------------- |
| `CRON_SECRET`         | Cleanup cron job     |
| `REDIS_URL`           | Resumable streams    |
| `TAVILY_API_KEY`      | Web search           |
| `EXA_API_KEY`         | Web search           |
| `FIRECRAWL_API_KEY`   | Web scraping         |
| `LANGFUSE_PUBLIC_KEY` | Observability        |
| `LANGFUSE_SECRET_KEY` | Observability        |
| `LANGFUSE_BASE_URL`   | Custom Langfuse URL  |
| `MCP_ENCRYPTION_KEY`  | MCP connectors       |

## Database Setup

```bash
bun db:migrate
```

<Note>
  For schema development workflows like Neon branching, see the [Database reference](/reference/database).
</Note>

## Run Development Server

```bash
bun dev
```

Open [http://localhost:3000](http://localhost:3000) to start building.
